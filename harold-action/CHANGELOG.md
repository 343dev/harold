# Changelog

Все значимые изменения в Harold Action будут документированы в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/),
и этот проект придерживается [Semantic Versioning](https://semver.org/lang/ru/).

## [Unreleased]

### Добавлено
- Первоначальная реализация Harold Action
- Автоматический анализ изменений размеров бандлов в Pull Request'ах
- Поддержка различных типов проектов (React, Vue, Angular, Next.js, etc.)
- Интеллектуальное кэширование для оптимизации производительности
- Параллельная обработка базовой и PR веток
- Система безопасности для fork репозиториев
- Early exit оптимизация для PR без изменений в бандле
- Подробные отчеты с разбивкой по файлам и категориям
- Настраиваемые пороговые значения для предупреждений
- Поддержка кастомных конфигураций Harold
- Автоматическое создание и обновление комментариев в PR
- Graceful fallback для ограниченных прав доступа
- Comprehensive test suite (unit, integration, E2E)

### Безопасность
- Автоматическое определение fork репозиториев
- Валидация команд сборки в fork'ах
- Маскирование секретов в логах
- Ограничения размера конфигурационных файлов
- Защита от path traversal атак
- Безопасная обработка GitHub токенов

## [1.0.0] - 2025-02-09

### Добавлено
- Первый стабильный релиз Harold Action
- Полная функциональность анализа размеров бандлов
- Документация и примеры использования
- Поддержка GitHub Marketplace

### Изменено
- Оптимизирована производительность для больших проектов
- Улучшена обработка ошибок и отчетность
- Обновлена документация с дополнительными примерами

### Исправлено
- Проблемы совместимости с различными версиями Node.js
- Ошибки обработки специальных символов в путях файлов
- Проблемы с кэшированием в Windows окружениях

### Безопасность
- Усилены проверки безопасности для fork репозиториев
- Добавлена дополнительная валидация входных параметров

## Типы изменений

- **Добавлено** - для новой функциональности
- **Изменено** - для изменений в существующей функциональности
- **Устарело** - для функциональности, которая скоро будет удалена
- **Удалено** - для удаленной функциональности
- **Исправлено** - для исправления ошибок
- **Безопасность** - в случае уязвимостей

## Соглашения о версионировании

Harold Action следует [Semantic Versioning](https://semver.org/lang/ru/):

- **MAJOR** версия увеличивается при внесении несовместимых изменений в API
- **MINOR** версия увеличивается при добавлении функциональности с обратной совместимостью
- **PATCH** версия увеличивается при исправлении ошибок с обратной совместимостью

## Миграция между версиями

### С версии 0.x на 1.0.0

Harold Action 1.0.0 полностью обратно совместим с предыдущими версиями. Никаких изменений в конфигурации не требуется.

Рекомендуется обновить версию в ваших workflow файлах:

```yaml
# Старая версия
- uses: harold-team/harold-action@v0.9.0

# Новая версия
- uses: harold-team/harold-action@v1.0.0
# или используйте latest
- uses: harold-team/harold-action@v1
```

### Новые возможности в 1.0.0

1. **Улучшенное кэширование**: Автоматическое кэширование снимков и результатов сборки
2. **Параллельная обработка**: Ускорение анализа за счет параллельной сборки веток
3. **Early exit**: Пропуск анализа для PR без изменений в бандле
4. **Расширенная безопасность**: Дополнительные проверки для fork репозиториев
